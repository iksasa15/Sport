# أكواد بايثون: أيها ضروري وأيها اختياري

## ✅ كل ملفات الباكند (`backend/`) مستخدمة

جميع الملفات في `backend/` يتم استيرادها أو استخدامها في التطبيق أو الاختبارات. **لا تحذف أي ملف من backend/**.

---

## ⚠️ سكربت مكرر (اختياري حذفه)

| الملف | السبب | البديل |
|-------|--------|--------|
| **`scripts/run_example.py`** | يفعّل نفس التحليل (رفع فيديو → تقرير) | استخدم: `python main.py --analyze "مسار/الفيديو.mp4" --sport football` |

لو حذفته، استخدم دائماً:
```bash
python main.py --analyze path/to/video.mp4 --sport football
```

---

## 📌 سكربتات مفيدة (لا تحذفها)

| الملف | الاستخدام |
|-------|-----------|
| **`download_model.py`** | تحميل نموذج Pose مسبقاً لتسريع أول تحليل |
| **`scripts/test_api.py`** | اختبار الـ API عندما السيرفر شغال |
| **`scripts/train_sport_module.py`** | تدريب/تحسين بيانات رياضة معينة |
| **`scripts/run_sport_test.py`** | تشغيل تحليل لرياضة من سطر الأوامر مع خيارات |

---

## الخلاصة

- **أكواد بايثون مالها داعي بالمعنى الحرفي:** لا يوجد؛ كل الـ backend مستخدم.
- **الوحيد المكرر:** `scripts/run_example.py` (نفس فكرة `main.py --analyze`).
- إذا حاب تبسط السكربتات: احذف `run_example.py` واستخدم `main.py --analyze` فقط.
