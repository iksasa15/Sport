# ملفات يمكنك حذفها لتنظيم المشروع

## ✅ آمن الحذف (يُعاد توليده أو غير ضروري)

### 1. `__pycache__` (كاش بايثون)
- **ماذا:** مجلدات فيها ملفات `.pyc`
- **الحذف:** آمن دائماً، بايثون يعيد إنشاءها عند التشغيل
```bash
find . -type d -name __pycache__ -exec rm -rf {} + 2>/dev/null
```

### 2. التقارير القديمة `reports/`
- **ماذا:** تقارير PDF, CSV, JSON من التحليلات السابقة (~17MB، أكثر من 1200 ملف)
- **الحذف:** آمن؛ أي تحليل جديد يولد تقارير جديدة
- **لو حاب تحتفظ ببعضها:** انسخ المجلد أو الملفات اللي تبيها ثم احذف الباقي
```bash
rm -rf reports/*
# أو احذف رياضة معينة فقط:
# rm -rf reports/Football/*
```

### 3. فيديوهات المخرجات `output/`
- **ماذا:** فيديوهات بعد رسم الهيكل العظمي (~69MB)
- **الحذف:** آمن؛ تُعاد إنشاؤها عند تحليل فيديو مع "live overlay"
```bash
rm -rf output/*
```

### 4. تقارير مكررة في `sports/Football/reports/`
- **ماذا:** تقارير قديمة في مسار ثاني (التقارير الرسمية تروح لـ `reports/Football/`)
- **الحذف:** آمن؛ التطبيق يعتمد على `reports/` وليس هذا المجلد
```bash
rm -rf sports/Football/reports/*
```

---

## ⚠️ انتبه قبل الحذف

### `uploads/`
- **ماذا:** فيديوهات رفعها المستخدم (~51MB)
- **الحذف:** يفقدك كل الفيديوهات المرفوعة. احذف فقط لو ما تحتاجها.

### `models/` (في جذر المشروع)
- **ماذا:** فيه `pose_landmarker_lite.task` (نموذج MediaPipe)
- **الحذف:** لا تحذفها؛ التطبيق قد يحتاجها للوضع (Pose).

---

## ملخص سريع

| المجلد/الملف | احذف؟ | ملاحظة |
|---------------|--------|--------|
| `**/__pycache__/` | ✅ نعم | كاش بايثون، يُعاد توليده |
| `reports/**` | ✅ نعم | تقارير قديمة، تُعاد بالتحليل |
| `output/*` | ✅ نعم | فيديوهات مخرجة، تُعاد |
| `sports/Football/reports/*` | ✅ نعم | نسخة مكررة من التقارير |
| `uploads/*` | ⚠️ حسب الحاجة | فيديوهات المستخدم |
| `models/` | ❌ لا | نموذج Pose |

---

## أمر واحد لتنظيف آمن (بدون uploads)

```bash
# من جذر المشروع Sport/
find . -type d -name __pycache__ -exec rm -rf {} + 2>/dev/null
rm -rf reports/*
rm -rf output/*
rm -rf sports/Football/reports/*
```

بعدها المشروع يبقى يعمل؛ التقارير والفيديوهات المخرجة تُعاد عند الاستخدام.
